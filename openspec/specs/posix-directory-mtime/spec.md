## ADDED Requirements

### Requirement: Parent directory mtime updated on child file addition
When a file is added to a directory, the system SHALL generate an additional "modified" `DiffEntry` for the parent directory with `change_time` set to the maximum `change_time` among all added files in that directory. Only one directory entry per unique parent directory SHALL be generated per round.

#### Scenario: Single file added to a directory
- **WHEN** a round contains one added file at path `/vol/share/user1/docs/report.docx` with `change_time = "2025-01-15T10:30:00Z"`
- **THEN** the system SHALL also emit a "modified" `DiffEntry` with `path = "/vol/share/user1/docs/"`, `type = "modified"`, `size = "0"`, and `change_time = "2025-01-15T10:30:00Z"`

#### Scenario: Multiple files added to same directory
- **WHEN** a round contains three added files in `/vol/share/user1/docs/` with change times T1, T2, T3 (where T3 is latest)
- **THEN** exactly one "modified" entry SHALL be generated for `/vol/share/user1/docs/` with `change_time = T3`

#### Scenario: Files added to different directories
- **WHEN** a round contains added files in `/vol/share/user1/docs/` and `/vol/share/user2/projects/`
- **THEN** one "modified" entry SHALL be generated for each unique parent directory

### Requirement: Parent directory mtime updated on child file deletion
When a file is deleted from a directory, the system SHALL generate an additional "modified" `DiffEntry` for the parent directory with `change_time` set to `Instant.MAX.toString()`. This is because the deletion itself carries `Instant.MAX` as the sentinel time, and the directory mtime should reflect that the deletion occurred with no specific timestamp.

#### Scenario: Single file deleted from a directory
- **WHEN** a round contains one deleted file at path `/vol/share/user1/docs/old_report.docx`
- **THEN** the system SHALL also emit a "modified" `DiffEntry` with `path = "/vol/share/user1/docs/"`, `type = "modified"`, `size = "0"`, and `change_time = Instant.MAX.toString()`

#### Scenario: Mixed additions and deletions in same directory
- **WHEN** a round contains both added and deleted files in `/vol/share/user1/docs/`
- **THEN** the directory SHALL still get exactly one "modified" entry. If any deletion occurred, the `change_time` SHALL be `Instant.MAX.toString()` (since the deletion has no meaningful timestamp). If only additions occurred, the `change_time` SHALL be the max child addition timestamp.

### Requirement: Directory mtime entries generated as post-processing step
The directory mtime entries SHALL be generated by a utility method `generateDirectoryMtimeEntries(List<DiffEntry> fileDiffs)` that accepts the list of file-level diff entries and returns a list of additional directory-level diff entries. This method SHALL be called at the end of every `evolve*` method in `FilesystemState` and every `generate*` method in `AttackGenerator`, and the returned entries appended to the result.

#### Scenario: Utility method extracts parent directories
- **WHEN** `generateDirectoryMtimeEntries` is called with diff entries containing paths `/vol/share/user1/docs/a.docx`, `/vol/share/user1/docs/b.docx`, `/vol/share/user2/logs/c.log`
- **THEN** it SHALL return entries for `/vol/share/user1/docs/` and `/vol/share/user2/logs/`

#### Scenario: No additions or deletions produces no directory entries
- **WHEN** `generateDirectoryMtimeEntries` is called with only "modified" type entries (no "added" or "deleted")
- **THEN** it SHALL return an empty list (directory mtime is not affected by content modifications in POSIX)

### Requirement: Directory entries have size zero
All directory-level "modified" diff entries SHALL have `size = "0"` to indicate they represent directory metadata, not file content.

#### Scenario: Directory entry size
- **WHEN** a directory "modified" entry is generated
- **THEN** its `size` field SHALL be `"0"`
