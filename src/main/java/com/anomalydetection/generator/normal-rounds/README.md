# 正常轮次基准数据 — 详细说明

## 概述

本目录包含 10 个标准快照差异 JSON 文件，模拟企业网络共享中真实用户的日常文件系统活动。这些数据用于基线校准和误报压力测试，是勒索软件异常检测器基准测试系统的一部分。

每个 JSON 文件代表一轮快照差异检测周期的正常活动。

---

## 变异轴框架

每轮活动由 6 个变异轴共同决定，确保覆盖广泛的正常行为分布：

| 轴 | 范围 | 说明 |
|---|---|---|
| **活动水平** | 2,800–22,000 操作 | 总操作量，对应 300K 文件系统的约 1%–7.3% |
| **操作类型比例** | 修改 36–62% / 重命名 5–15% / 新增 18–38% | 增/删/改/重命名的混合比例 |
| **活跃用户** | 3–10 人（共 10 用户） | 非活跃用户在本轮无操作 |
| **时间窗口** | 2.5–12 小时 | 操作分布在长短不同的时间跨度内 |
| **非工作时间比例** | 8–85% | 操作落在 18:00–23:59 或 05:00–07:59 的比例 |
| **活动波动类型** | quiet / normal / busy | quiet 低量，normal 适中，busy 包含突发 |

## JSON 格式说明

每个文件遵循以下结构（详见 `benchmark.md`）：

```json
{
  "diffs": [
    {"path": "/vol/share/user1/docs/file_0042.docx", "type": "modified", "size": "56832", "change_time": "2026-04-22T10:30:00Z"},
    {"path": "/vol/share/user3/logs/file_0100.log",   "type": "deleted",  "size": "2048",   "change_time": "2026-04-22T10:31:00Z"},
    {"path": "/vol/share/user5/temp/file_added_42.tmp","type": "added",   "size": "4096",   "change_time": "2026-04-22T10:32:00Z"}
  ],
  "summary": {"files_added": 1, "files_modified": 1, "files_deleted": 1}
}
```

### 字段说明

| 字段 | 类型 | 说明 |
|---|---|---|
| `diffs[].path` | string | 文件完整路径，格式：`/vol/share/user{N}/{dir}/file_{XXXX}.{ext}` |
| `diffs[].type` | string | 操作类型：`added`（新增）、`modified`（修改）、`deleted`（删除） |
| `diffs[].size` | string | 操作**后**的文件大小（字节数，字符串形式） |
| `diffs[].change_time` | string | ISO-8601 UTC 时间戳 |
| `summary.files_added` | int | added 类型操作总数 |
| `summary.files_modified` | int | modified 类型操作总数 |
| `summary.files_deleted` | int | deleted 类型操作总数 |

### 重命名操作

重命名在快照差异中以 **delete + add 对** 表示：
- 一条 `deleted` 记录指向原路径
- 一条 `added` 记录指向带版本后缀的新路径

版本后缀包括：`_v2`, `_v3`, `_new`, `_backup`, `_old`, `_final`, `_copy`, `_draft`

### 文件大小分布

| 概率 | 大小范围 | 类型 |
|---|---|---|
| 70% | 1 KB – 100 KB | 小文件（文本、配置、日志） |
| 20% | 100 KB – 10 MB | 中等文件（文档、图片） |
| 9% | 10 MB – 100 MB | 大文件（媒体、数据库） |
| 1% | 100 MB – 1 GB | 超大文件（归档、备份） |

修改操作的大小变化区间为 -30% 到 +30%（均匀分布），重命名操作变化 ±5% 或保持不变。

## 生成方式

这 10 个文件由 `NormalRoundGenerator.java`（位于上级 `generator/` 目录）自动生成，使用确定性随机种子 `42L`。如需重新生成，运行：

```bash
javac -d target/classes src/main/java/com/anomalydetection/generator/NormalRoundGenerator.java
java -cp target/classes com.anomalydetection.generator.NormalRoundGenerator path/to/normal-rounds
```
